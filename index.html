Video tutorials for PPTAM - Tools for Production and Performance Testing Based Application Monitoring.

PPTAM can be installed in various ways. Here is the installation within a Virtual Box using Vagrant.
The command 'vagrant up' within the installation folder installs two machines: the driver and the testbed machine.

Once both machines are intalled, we can download PPTAM into the driver.
Login to the driver using 'vagrant ssh driver', and execute the script '/vagrant/scripts/get_pptam.sh'.

Experiments are configured in separate folders (that we always prefix with 'design_'). There you can setup the configuration in configuration.ini and then define a configuration for each test in test_plan.ini. configured

To then run the experiments, you change to the folder 'execute' and run 'execute_test.py' passing the folder with the configuration as the first parameter.

The collected data is currently collected through:
  • https://locust.io
  • docker stats
  • https://www.jaegertracing.io
It is stored within Influx DB, accessible through a redirected port to the driver machine, at http://localhost:9999 using the user name 'pptam' and the default password '12345678'.
After logging in, the bucket containing the collected data is named 'pptam'.
Within that bucket, queries can be created with the data explorer or dashboards can be created with the boards feature of influx db.
